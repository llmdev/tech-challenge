## Estrutura do monorepo

```
packages/
  @repo/button       → Botão com variantes, tamanhos e suporte a ícones
  @repo/input        → Campo de texto com label e estado de erro
  @repo/select       → Seletor acessível baseado em Radix UI
  @repo/navbar       → Barra de navegação superior
  @repo/sidebar      → Navegação lateral responsiva
  @repo/balance-card → Card de saldo para contexto financeiro
  @repo/statement    → Linha de extrato com tipo, valor e data
  @repo/icons        → Ícones SVG via factory createIcon
  @repo/theme        → Plugin Tailwind com tokens de cor e modo escuro

apps/
  web                → App Next.js 15 (consumidor principal)
  storybook          → Este Storybook (documentação viva)
```

Cada pacote exporta diretamente o source `.tsx` — sem etapa de build. O transpile acontece no consumidor (`next.config.ts` ou Vite).

## Tokens de tema

O tema é construído sobre variáveis CSS HSL injetadas pelo `defaultThemePlugin` do Tailwind. As classes semânticas mapeiam sempre para esses tokens:

## Como contribuir

O Design System cresce por contribuições incrementais. Cada PR deve ser atômica: um componente, uma correção ou um token — não os três juntos.

### Configuração inicial

```bash
git clone <repo>
pnpm install          # instala todas as dependências do monorepo
pnpm dev              # sobe web (localhost:3000) + storybook (localhost:6006) em paralelo
```
